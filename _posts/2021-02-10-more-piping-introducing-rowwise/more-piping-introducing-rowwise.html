I said the default behaviour of mutate() and summarise() is to work on variables, i.e.Â vectors, whether that is to work on all the values of the variable if there is no group_by(), or within the groups if there is a grouping. If I want to do something on individual values, i.e.Â by rows, â€œrowwiseâ€, then I have to use rowwise() which basically treats each row as a group.

If, as you often will in that situation, you want to use a function of more than one value, i.e.Â values from more than one variable, then you have to remember to use c_across() now, not across(): â€œc_â€ as itâ€™s by column.

You also have to remember to ungroup() after any mutate() as you probably donâ€™t want future functions to handle things one row at a time.

getAlphaForBoot <- function(dat, i) {
  # a little function that happens to use psych::alpha to get alpha
  # but indexes it with i as boot() will require
  psychometric::alpha(na.omit(dat[i,]))
}
getAlphaForBoot(tmpDat[, 3:7], 1:nrow(tmpDat)) # just checking that it works
[1] 0.9097275
Show code 
bootReps <- 1000
getCIAlphaDF3 <- function(dat, ciInt = .95, bootReps = 1000) {
  tmpRes <- boot::boot(na.omit(dat), getAlphaForBoot, R = bootReps)
  tmpCI <- boot::boot.ci(tmpRes, conf = ciInt, type = "perc")$percent[4:5]
  return(data.frame(alpha = tmpRes$t0,
                    LCL = tmpCI[1],
                    UCL = tmpCI[2]))
}
